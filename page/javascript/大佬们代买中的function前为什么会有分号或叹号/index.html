# 为什么要有";"和"!"，什么时候要用它们？

（吐槽：总觉得";"和信仰有关，因为总有一些极端分子不写";"） 上面的逻辑很容易看出来，两行代码，如果没有分号就会被浏览器拼到一起去执行，除非有个浏览器能识别出的关键字，比如说 let/var，才知道这是新的语句。 但很多情况一个 js 可能会有多个人编辑，前面的代码有其别人编写，如果没写";"，我们又不能修改别人的代码，这时我们就需要在我们这行补一个";"分号了。 而叹号"!"的作用一般有三个，除了大家经常用的非运算和转换 boolean 型以外，还有一个就是表达式转换，就是把一个非表达式转换成表达式。 function(){}() 并不是一个合法的算数式，所以需要"!"来模拟(function(){})()的执行。

# 最后一个问题，关于打包

谈若上一行代码结尾没有分号";"，而首行用叹号"!"呢？ 其实这也是没问题的，因为!行已经被转换成表达式，而表达式是可以直接执行的。 但在建议上不要这么写，因为在打包的时候两行代码是要拼在一起，这样很容易报错，这也就是为什么大佬们在写代码要在行首添加分号";"了。
